The task is to be able to center a div or a text horizontally and/or vertically

1. Position Property in CSS

They are 4 positions:

  • Absolute
  • Relative
  • Fixed
  • Sticky

These positions needs to set TOP, LEFT, RIGHT or BOTTOM

Absolute

Absolute positioning of the child needs a relative parent, if it does not have an ancestor element set as position relative it will take the view port. Absolute positioning will take the closest relative in case of several relative ancestor. Absolute relative block match the size of their contents instead and the width in % will be relative to their ancestor. Usually don't use position absolute too often.

Relative

Relative positioning on the child behaves much like the absolute when it is relative to his parent container. Position relative is by default relative to his parent container. Position relative, the space on the DOM is still reserved. Position relative open z-index for staking contents.

Fixed

Like position absolute it shrinks to his content so it requires sometimes width:100%. The other contents ignores it like position absolute. They dont know it exists. Once defined the TOP, LEFT, RIGHT or BOTTOM it will stay in the fixed position all the time even when we scroll the page. It is not really used as a header now.

Sticky

Regular block element. Live in the flow compared to Fixed and Absolute. The TOP value is important as it is where it starts to sticks. Defining TOP:0 for instance will make the element sticks to the top of his relative container or view port. While scrolling down, it will stick until it disappears with his container. It will not overflow. Position sticky does not shrink but will not overflow also. So sometimes it is not sticking with display:Flex. With Flex they have by defaut align-items: strech. Therefore the sticky can't scroll down and the align-self should be setup to start
